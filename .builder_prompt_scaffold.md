# MVP Plan — uia-agent

## 1. The minimum demonstrable product

A pip-installable Python CLI that, given one natural-language sentence plus a running Windows application, drives that application through real UIA clicks and keystrokes to completion — no scripts, no selectors, no templates. The 60-second demo: a user opens Notepad, runs `uia-agent run --app Notepad "type a haiku about Windows 95, save it as poem.txt to the desktop"`, and watches the agent enumerate the UIA tree, type the haiku, open File → Save As, navigate the dialog, name the file, and click Save. The product is for developers building AI agents who need a Windows-thick-client action space — same audience as browser-use, different surface. Free, OSS, MIT — no paid tier in v0.1.

## 2. Core data model / primitive

The primitive is the **UIA Action Frame**: a pruned, LLM-readable snapshot of the Windows UI Automation tree of a single focused process, plus a typed action vocabulary that maps cleanly back to UIA control patterns.

```python
class UIANode(BaseModel):
    id: str                        # stable hash of (control_type, name, automation_id, depth_path)
    role: str                      # Button | Edit | MenuItem | ListItem | ...
    name: str | None               # visible label
    value: str | None              # current text/value if Value pattern supported
    enabled: bool
    bbox: tuple[int, int, int, int]
    patterns: list[str]            # ["Invoke", "Value", "ExpandCollapse", ...]
    children: list["UIANode"]

class Action(BaseModel):
    kind: Literal["click", "type", "select", "expand", "key", "wait", "done"]
    target_id: str | None          # references UIANode.id
    text: str | None               # for type/key
    reason: str                    # LLM's chain-of-thought trace, one sentence
```

The novelty isn't the data structure itself; it's treating the UIA tree as a first-class **action space** for an LLM loop — distinct from DOM (browser-use), pixels (Operator-style VLM), or scripted selectors (UiPath). The pruning rules (drop invisible / off-screen / unnamed / disabled nodes; cap tree to N tokens) are the defensible craft.

## 3. Minimum happy path

1. `pip install uia-agent` (≤ 20s on a typical Windows machine; pulls `uiautomation` + `pywinauto` + one LLM SDK).
2. `export ANTHROPIC_API_KEY=...` (or OpenAI) — ≤ 10s.
3. `uia-agent run --app Notepad "type 'hello world' and save it as hello.txt on the desktop"` — ≤ 5s to start.
4. Watch the terminal stream each `Action` the LLM chose plus its `reason`; the agent stops on `kind="done"` or after a max-step budget. Total runtime ≤ 60s for the demo task.
5. Inspect `hello.txt` on the desktop. First value delivered.

Total user effort ≤ 90 seconds from `pip install` to artifact on disk.

## 4. Architecture

Three modules in one process, one binary entry point:

```
+--------------------------------------------------+
|  cli.py  (Typer)                                 |
|     run  --app <name> "<instruction>"            |
|     dump --app <name>  (prints tree, debug aid)  |
+----------------+---------------------------------+
                 |
        +--------v---------+      +-----------------+
        |  agent.py        |<---->|  llm.py         |
        |  step-loop:      |      |  (Anthropic /   |
        |  observe→think   |      |   OpenAI shim)  |
        |  →act→verify     |      +-----------------+
        +----+-----+-------+
             |     |
+------------v+  +-v---------------+
| uia_tree.py |  | actions.py      |
| (snapshot + |  | (Invoke / Value |
|  pruning)   |  |  / SetText /    |
+-------------+  |  SendKeys /     |
                 |  Expand)        |
                 +-----------------+
```

No services, no daemon, no IPC. The agent loop is a plain Python `while` over (snapshot UIA → render to JSON → call LLM → parse `Action` → dispatch via `actions.py` → loop). LLM provider is pluggable behind a thin shim; the v0.1 ships with Anthropic + OpenAI.

## 5. Build milestones (each ≤ 8 hours)

### m1_dump_uia_tree (~8h)
`uia-agent dump --app Notepad` emits a pruned JSON tree of the focused Notepad window in under 1 second. Pruning rules wired: invisible / off-screen / unnamed-leaf nodes dropped, max depth 12, max nodes 400. Unit tests assert pruned tree fits in <8k tokens for Notepad, Calculator, and File Explorer. **Done** = `uia-agent dump` runs end-to-end against 3 stock Windows apps.

### m2_execute_actions (~12h)
`uia-agent run --app Notepad "<instruction>"` completes the observe→think→act loop. `actions.py` implements 7 action kinds against real UIA control patterns. The LLM step uses structured output (Anthropic tool use / OpenAI JSON-schema) so action parsing can't fail soft. Max-step budget = 25, on overshoot the CLI exits non-zero with the partial trace. **Done** = the haiku-into-Notepad demo from §1 succeeds on a clean Windows 11 VM.

### m3_demo_winforms (~12h)
